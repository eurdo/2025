# python實現雙向循環鏈表基本結構及其基本方法

11224215 顏玉棠

雙向循環鏈表是在雙向鏈表的基礎上發展的，雙向鏈表的最後一個節點指向起始節點，起始節點的上一個節點指向最後一個節點，就得到雙向循環鏈表。

雙向循環鏈表比雙向鏈表具有更多的優勢，節點的增加和刪除有很多優化的地方，從起點開始不必循環完整個鏈表就可以增加或刪除節點。

*雙向循環鏈表的優點

1.可從任一節點開始遍歷：
由於結構是循環的，可以從任何節點出發，順時針或逆時針遍歷整個鏈表。

2.插入與刪除更靈活：
在任意位置插入或刪除節點時，不需要像單向鏈表那樣只能從前到後慢慢找。
可以根據需要從前或從後進行操作，效能更佳。

3.節省尋找尾端或起點的時間：
尾端透過 head.prev 就可以找到，不用從頭開始一直找。

4.適合用於需要循環訪問的場景：
例如：音樂播放清單、排班系統、多人遊戲中的角色輪替等。

*常見應用場景

| 應用場景       | 為何適合用雙向循環鏈表？               |
| ---------- | -------------------------- |
| 音樂播放器播放清單  | 播放到最後一首後可自動回到第一首           |
| 排班系統       | 員工排班可循環輪替，每人當完班後自動排到下一輪    |
| 多人回合制遊戲    | 玩家結束回合後，自動輪到下一位，最後一位後回到第一位 |
| 網頁瀏覽器前進／後退 | 可透過 `prev` 和 `next` 快速切換頁面 |

*與單向／雙向鏈表比較

| 功能／特性    | 單向鏈表 | 雙向鏈表 | 雙向循環鏈表 |
| -------- | ---- | ---- | ------ |
| 有前後指標    | ❌    | ✅    | ✅      |
| 可以往前移動   | ❌    | ✅    | ✅      |
| 尾節點指向頭節點 | ❌    | ❌    | ✅      |
| 適合循環處理   | ❌    | 一般   | ✅      |
| 插入／刪除效率  | 普通   | 好    | 最佳     |
